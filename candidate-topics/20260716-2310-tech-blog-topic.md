- 검색 의도: 운영 중인 PostgreSQL에서 대량 삭제를 수행할 때 롱락과 bloat를 피하는 구체적 방법과 단계별 명령어를 알고 싶다.
- 실무 가치: 서비스 중단 없이 삭제 작업을 분할/백그라운드화하고 VACUUM/REINDEX 전략을 적용해 실무에서 즉시 적용 가능한 절차를 제공한다.
- 최신성 포인트: PostgreSQL 버전별 개선된 병렬 처리와 VACUUM 동작 차이가 결과에 영향을 줄 수 있어 현재 사용 중인 버전 문서 확인이 필요하다.
- 확인 권장 출처: PostgreSQL 공식 문서, 버전별 릴리스 노트, 운영 DB 벤더 가이드

### 3. Linux 시스템에서 디스크 사용량이 갑자기 늘었을 때 원인을 빠르게 진단하는 체크리스트

- slug: `linux-disk-usage-spike-diagnosis`
- 추천 이유: 운영 서버의 디스크 부족은 즉시 서비스 장애로 이어지므로 빠르게 원인을 좁히는 절차가 필요하다.
- 검색 의도: 디스크 사용량이 급증했을 때 어떤 명령과 로그를 우선 확인하고 어떤 조치를 취해야 하는지 단계별로 알고 싶다.
- 실무 가치: 실무에서 빠르게 루트 원인을 찾고 임시 조치(로그 회전, 파일 삭제, inode 점검)를 통해 서비스 복구를 가능하게 한다.
- 최신성 포인트: 파일시스템별(overlay, ext4, xfs) 동작과 시스템 유틸리티 버전 차이가 진단 결과에 영향을 줄 수 있어 환경별 문서 확인이 필요하다.
- 확인 권장 출처: Linux man 페이지, systemd 로그 문서, 파일시스템 및 배포판 운영 가이드
- 썸네일: 디스크 스파이크 진단 / 로그, 열린 파일, 삭제된 파일 확인 순서
