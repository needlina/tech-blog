### 1. Sigstore와 cosign으로 컨테이너 이미지·아티팩트 서명 및 CI 통합하기

- slug: `sigstore-cosign-ci-image-signing`
- 추천 이유: 소프트웨어 공급망 공격이 증가하면서 이미지와 배포 아티팩트의 서명·검증이 CI 파이프라인 기본 요소가 되었기 때문에 실무 도입 사례와 문제점을 정리하면 가치가 높다.
- 검색 의도: CI에서 cosign을 이용해 이미지 서명·검증을 자동화하는 방법과 키 관리, Rekor/fulcio 연동 실무 이슈를 알고 싶다.
- 실무 가치: 빌드 파이프라인 수정, 서명 키 롤오버 절차, 클러스터에서의 검증 자동화 예시 등 바로 적용 가능한 체크리스트와 스크립트를 제공한다.
- 최신성 포인트: Sigstore 생태계와 cosign 도구가 빠르게 발전하고 있어 최신 버전의 서명 형식·정책 변화가 배포 파이프라인에 즉시 영향을 준다.
- 확인 권장 출처: 공식 문서, 릴리즈 노트, 벤더 블로그, 표준 권고
- 썸네일: 사이프체인 서명 파이프라인 / cosign/Sigstore로 빌드부터 배포까지 아티팩트 신뢰성 확보

### 2. 패스키(WebAuthn) 도입 체크리스트 — 서버 검증과 기존 계정 마이그레이션

- slug: `webauthn-passkeys-server-migration-checklist`
- 추천 이유: 패스키·WebAuthn 채택이 급증하면서 인증 UX 개선과 동시에 서버 측 검증·레거시 계정 통합에서 실무 이슈가 많이 발생한다.
- 검색 의도: 서버에서 WebAuthn 어설션을 검증하는 방법, 브라우저/플랫폼 호환성, 기존 비밀번호 계정과 병행 운영하는 전략을 찾고 있다.
- 실무 가치: 인증 흐름 설계, 클라이언트옵션-레플백 처리, 계정 복구·2차 인증 정책, 라이브러리 선택 기준 등 실무 적용 가능한 지침을 제공한다.
- 최신성 포인트: 주요 플랫폼의 패스키 지원 범위와 표준 해석이 짧은 주기로 업데이트되어 도입 설계에 최신 호환성 확인이 필요하다.
- 확인 권장 출처: 표준 문서(WebAuthn), 브라우저 벤더 문서, 플랫폼 가이드
- 썸네일: 패스키 전환 가이드 / 비밀번호에서 WebAuthn 기반 패스키로 안전하게 이동하기

### 3. 쿠버네티스에서 dockershim 제거 이후 컨테이너 런타임 마이그레이션 체크리스트

- slug: `k8s-dockershim-runtime-migration-checklist`
- 추천 이유: 클러스터 런타임 전환은 운영 중단과 호환성 문제를 일으킬 수 있어 마이그레이션 전략과 검증 포인트를 정리하면 실무에 도움이 된다.
- 검색 의도: 이미지 빌드·레지스트리, 로그/모니터링, privileged 컨테이너, CNI/CSI 호환성 등 런타임 전환 시 체크해야 할 항목을 알고 싶다.
- 실무 가치: 호환성 테스트 계획, node 교체 패턴, 데몬셋·CRI 로그 관찰법, 도구(ctr/crictl) 사용법 등 당장 실행 가능한 마이그레이션 단계별 지침을 제공한다.
- 최신성 포인트: 쿠버네티스 생태계에서 크루타임 표준화와 관련 도구 업데이트가 지속되어 현재 구성에 따른 영향 분석이 바로 필요하다.
- 확인 권장 출처: 쿠버네티스 공식 문서, 런타임 릴리즈 노트, 벤더 마이그레이션 가이드
- 썸네일: 런타임 전환 준비 / Docker 기반 워크로드를 containerd/CRI-O로 옮길 때의 실무 점검항목

### 4. 경량화·양자화된 LLM 배포 실무: GGUF/GGML, ONNX, Triton 비교와 톱니바퀴

- slug: `llm-quantization-gguf-onnx-triton-practical`
- 추천 이유: 대형 언어 모델을 실제 서비스에 배포하려면 양자화 포맷과 추론 런타임 선택, 정확도-성능 트레이드오프를 이해하는 것이 중요하다.
- 검색 의도: GGUF/GGML 같은 경량 포맷과 ONNX/Triton 등의 런타임에서의 성능·호환성 차이, 양자화 시 품질 저하를 줄이는 방법을 찾고 있다.
- 실무 가치: 양자화 파이프라인, 모델 변환 체크리스트, 메모리·GPU 오프로드 전략, 프로덕션 모니터링 지표를 통한 성능 검증법을 제공한다.
- 최신성 포인트: 오픈소스 추론 스택과 양자화 툴이 빠르게 업데이트되어 배포 환경별 최적 조합을 지금 확인해야 비용과 응답성 손실을 막을 수 있다.
- 확인 권장 출처: 오픈소스 프로젝트 문서, 런타임 릴리즈 노트, 벤더 블로그
- 썸네일: 경량 LLM 배포 가이드 / 모델 양자화 포맷과 런타임 선택으로 비용·지연 줄이기

### 5. CI/CD에서 OIDC 워크로드 아이덴티티로 클라우드 자격증명 관리하기

- slug: `ci-oidc-workload-identity-cloud-credentials`
- 추천 이유: 장기적 크리덴셜을 제거하고 CI에서 클라우드 리소스에 안전하게 접근하는 패턴으로 OIDC 연동이 보편화되어 실무에서 도입 사례와 주의점을 정리할 필요가 있다.
- 검색 의도: GitHub Actions/GitLab CI에서 OIDC 토큰을 발급받아 AWS/GCP/Azure의 권한을 위임하는 설정, 토큰 수명·audience 설정과 위험요소를 알고 싶다.
- 실무 가치: OIDC 신뢰관계 구성 예제, 최소권한 역할 매핑, 토큰 오디언스·리플레이 방지 설정, 증명서 유효성 점검 등 바로 적용 가능한 운영 팁을 제공한다.
- 최신성 포인트: 클라우드 벤더와 CI 플랫폼의 OIDC 지원 정책과 권장 설정이 계속 바뀌어 현재 권장 구성으로 보안 사고를 예방해야 한다.
- 확인 권장 출처: 클라우드 제공업체 문서, CI 플랫폼 공식 가이드, 보안 권고
- 썸네일: CI→클라우드 무비밀번호 인증 / OIDC를 이용해 GitHub/GitLab 액션에서 클라우드 권한을 안전하게 부여하는 방법
